I have refined the plan to incorporate the "Vintage/Cute" design aesthetic while strictly adhering to the **Tabler (Bootstrap 5)** requirement and **NO Tailwind CSS** rule.

### **Phase 1: Database Architecture & Foundation (Core)**
**Goal:** Robust data structure for Roles, Permissions, and History.
1.  **Refine Migrations:**
    *   **Users:** Add `role_id`, `is_active`, `deleted_by`, `created_by`, `last_login_ip`.
    *   **Roles & Permissions:** Dynamic menu access control (Superadmin, Admin, Owner, User).
    *   **Places (`tempat_makan`) & Proposals (`usulan_tempat`):** Include Google Place ID, Ratings, Prices, Location, Status.
    *   **Audit Logs:** Track who did what (Requirement #8).
    *   **Settings:** App config (Logo, Contact, etc.).
2.  **Models:** Implement `SoftDeletes` and `RecordActivity` traits for all models.

### **Phase 2: Authentication & User Management (Tabler UI)**
**Goal:** Secure, Vintage-themed Auth.
1.  **Auth Views:** Custom Login/Register pages using Tabler + Custom CSS for "Vintage" look (Warm colors, retro fonts).
2.  **Verification:** Email activation flow + Captcha (Offline Math & Online Google).
3.  **Superadmin Features:**
    *   Restore/Soft Delete Users.
    *   Role & Menu Access Management.
    *   Discord Webhook for critical alerts.

### **Phase 3: Dining Place Proposal Workflow**
**Goal:** User -> Admin -> Live flow.
1.  **Forms:** "Add Place" form using Tabler components, styled to look friendly/cute.
2.  **Logic:**
    *   **User:** Submit proposal (can edit if pending).
    *   **Admin:** Validate -> Approve (moves to `tempat_makan`).
    *   **Edit:** Admin fixes typos before approval.

### **Phase 4: Frontend & "Vintage" Design (The Look)**
**Goal:** "Lucu" (Cute) & Vintage aesthetic using Tabler + Custom CSS/JS.
1.  **Design System:**
    *   **Base:** Tabler (Bootstrap 5).
    *   **Theme:** "Vintage/Cute" -> Custom CSS for warm color palette (cream, pastel, brown), rounded corners, and playful interactions.
    *   **Layout:** Sidebar + Header Bar (Fixed/Sticky).
2.  **Guest Logic:**
    *   Homepage: Show "Recommended Places" cards.
    *   **Restriction:** Clicking a list item -> Redirects to Login/Register if not authenticated.
3.  **Maps:**
    *   Google Maps JS API configured to **only show places of type 'restaurant'**.
    *   Custom markers to fit the vintage theme.

### **Phase 5: Owner Dashboard & Reporting**
**Goal:** Business insights.
1.  **Visuals:** Tabler Charts (ApexCharts) for User Growth, Revenue, etc.
2.  **Reports:** Monthly/Yearly/Weekly breakdowns.
3.  **Export:** Backup/Restore database functionality.

**Technical Stack:**
*   **Backend:** Laravel 11.
*   **Frontend:** Tabler (Bootstrap 5) + Custom "Vintage" CSS/JS.
*   **Map:** Google Maps JS API.
